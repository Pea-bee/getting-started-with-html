The Basics of the Web and HTML

The World Wide Web (or the web)

First we define a few terms:-

A web browser is a program that runs on your computer to interpret and display files from the web on your computer.
The internet consists of the physical lines and routers to form the world's largest computer network.
Servers are the computers which host the files that make up the web.
HTTP is the main protocol that broswers use to communicate with the servers.

The web is essentially a collection of HTML and other documents, which are shared via the internet. When you visit a web page, such as www.udacity.com, your computer - via a web browser - transmits a HTTP Request to a server over the internet. The links between these web pages, called hyperlinks, give the web its weblike structure.

This video provides a good explanation.

HTML

HTML stands for Hypertext Markup Language. HTML documents are the basis for almost all web pages and form the majority of the content on the web.

HTML is made up of :-

- text content, i.e. "what you see";
- markup, i.e. "how it looks";
- references to other documents, such as images and videos; and
- links to other pages.

This video provides a good overview.

HTML Markup

HTML documents consist of a series of HTML elements, and when writing HTML we use HTML tags to tell the browser exactly which element is to be rendered on the page. The tags essentially seperate normal text from the HTML code and allow all the interesting stuff such as images and videos. Tags can be closed, e.g. when making type bold, or void, e.g. when displaying images.

This video gives a good description.

Computers are Stupid

Programmers needs to write in a language the computer understands, and because computers interpret instructions literally, a small error by a "smart" programmer can lead to huge problems in a program due to a "stupid" computer.

Inline v Block

HTML elements can be either inline or block. Block elements form an "invisible box" around the content, which allows attributes such as height and width. Inline elements occupy only the space bounded by the tags which contain the content. Examples of such elements are line breaks and anchors.

Span v Div

There exists ways to attach styles to content which allow us to adjust the behaviours of how they display. Span is inline while Div is block.
